Qiskit Runtimeを使用する際の責任範囲について

Qiskit Runtimeを使用する際のお客様およびIBM®の具体的な責任範囲については、以下の各セクションをご確認ください。全体的な利用規約については、IBM Cloud®の利用規約と注意事項をご参照ください。

インシデントおよびオペレーション管理​

インシデントおよびオペレーション管理には、監視、イベント管理、高可用性、問題の特定、復旧、完全な状態のバックアップと復旧などのタスクが含まれます。

タスクIBMの責任お客様の責任
インフラストラクチャの健全性管理
  • 新機能、アップデート、バグ修正を必要に応じて透明性を持って継続的に提供します。
  • 計画されたメンテナンス、セキュリティ情報、または計画外の停止について通知します。
  • サービス通知に関するメールを受け取るよう通知設定を行ってください。
  • 一般的なお知らせについてはIBM Cloudステータスページを監視してください。
インフラストラクチャの健全性監視と通知
  • インフラストラクチャ環境のすべての問題を修正します。
  • 該当するインシデントについてお客様に通知します。
  • 報告された各インシデントの影響を評価してください。
  • 必要に応じてIBMサポートにお問い合わせください。
IBM Cloudの高可用性SLAの維持IBM Cloud パブリック サービス レベル アグリーメント（SLA）に基づいてQiskit Runtimeを運用します。高可用性のベストプラクティスに従ってください。

変更管理​

変更管理には、デプロイ、設定、アップグレード、パッチ適用、設定変更、削除などのタスクが含まれます。

タスクIBMの責任お客様の責任
メジャーバージョンのアップグレードメジャーバージョンのアップデートに必要な可用性とツールを提供します。旧バージョンのサポート終了前に、サポート対象バージョンへ移行してください。
APIバージョンの変更REST APIのバージョン管理を行います。必要なバージョンのREST APIと統合してください。
アップデート、修正、および新機能
  • 継続的デリバリーモデルに従い、定期的なアップデート、バグ修正、新機能を透明性を持って提供します。
  • お客様に影響を与える変更についての通知を掲載します。
  • メール通知を受け取るよう設定し、一般的なお知らせについてはIBM Cloudステータスページを監視してください。
  • システム生成の通知およびプラットフォームのお知らせを確認し、適切な対応を取ってください。

ID・アクセス管理​

ID・アクセス管理には、認証、認可、アクセス制御ポリシーの管理、アクセスの承認・付与・取り消しなどのタスクが含まれます。

タスクIBMの責任お客様の責任
IDとアクセス制御IBM Cloudを通じてリソースへのアクセスを管理・制限する機能を提供します。Cloud Identity and Access Management（IAM）を通じてリソースへのアクセスを付与・取り消し・管理してください。
サービス認証認証済みのユーザーのみがQiskit Runtimeのサービスおよびリソースにアクセスできるようにします。
  • 不正アクセスからの保護のため、APIキーを安全に管理してください。
  • セキュリティポリシーの要件に従ってAPIキーをローテーション・更新してください。
オブザーバビリティユーザーのアクションを監査するためのIBM Cloud Activity Tracker Event Routingとの統合を可能にします。ユーザーアクティビティを追跡するために、IBM Cloud Activity Tracker Event Routingまたはその他の機能を設定してください。

セキュリティおよび規制準拠​

セキュリティおよび規制準拠には、セキュリティ管理の実装およびコンプライアンス認証などのタスクが含まれます。

タスクIBMの責任お客様の責任
全般
  • 様々な業界コンプライアンス標準に見合った管理策を維持します。
  • サービス機能の監視、分離、および復旧を行います。
  • サービスインターフェースの健全性を監視・報告します。
個人情報や規制対象データが提供されないようにしてください。
脆弱性脆弱性およびセキュリティコンプライアンスの問題を検出するための監視を行います。
  • 潜在的な修正措置を提供するセキュリティ情報を通じて、脆弱性やセキュリティ問題について常に情報を収集してください。
  • QiskitのGitHubリポジトリのセキュリティセクションを確認してください。すべてのセキュリティ上の脆弱性はそこに掲載されます。
  • IBM Cloudステータスウェブサイトを確認し、IBM Cloudプラットフォーム、インフラストラクチャ、および主要サービスに影響する重要なイベントに関するお知らせやセキュリティ情報を確認してください。
暗号化公衆ネットワーク上および Cloud Service のプライベートデータセンターネットワーク内で転送されるデータを暗号化します。HTTPSやTLSなどの暗号化方式を使用してデータの転送中の保護を行います。必要に応じて、接続がエンドツーエンドで暗号化されていることを確認してください。
セキュリティ
  • インフラストラクチャ内の保存データおよび転送中のデータのセキュリティを確保します。
  • 主要な標準に準拠したセキュアなサービスを提供します。
適切なリソースおよびリソースグループへのユーザーアクセスを制限してください。

災害復旧​

タスクIBMの責任お客様の責任
全般影響を受けたリージョンのプロビジョニング環境とリソースを復元または再構築します。
  • インスタンスとリソースの状態を監視してください。
  • 利用可能な場合は、別のリージョンに新しいQiskit Runtimeサービスインスタンスをプロビジョニングしてください。
サービスの復旧Qiskit Runtimeサービスの機能を自動的に復旧・復元し、通常の運用状態に戻します。復旧後、影響を受けたジョブを再送信してください。
ユーザー設定データのバックアップIBMはユーザー設定データのバックアップについては責任を負いません。必要に応じて設定データをバックアップしてください。
ワークロードのバックアップと復旧IBMはユーザーのワークロードデータのバックアップまたは復旧については責任を負いません。必要に応じてワークロードのバックアップと復旧を行ってください。
