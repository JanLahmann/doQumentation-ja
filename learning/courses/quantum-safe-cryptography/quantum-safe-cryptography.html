量子安全暗号入門​

量子コンピュータは、現在普及している古典コンピュータとは根本的に異なる、より強力な計算パラダイムを体現しているため、量子コンピュータが普及した世界を想定して、暗号セキュリティの再評価が必要です。量子コンピュータは、古典コンピュータでは不可能な新たな暗号攻撃を可能にします。

こうした攻撃は、現在送受信・保存されているデータに対して将来実施される可能性があります。これは「今収集して後で復号（harvest now, decrypt later）」として知られる手法です。そのため、必要なシステムが実用化されるまで待つのでは不十分であり、今すぐ対策を講じる必要があります。

量子安全暗号（QSC）の分野は、量子コンピュータと古典コンピュータ両方からの攻撃に耐えられる暗号方式の特定と開発に取り組む研究領域です。これは量子耐性暗号またはポスト量子暗号とも呼ばれます。

以前のレッスンでは、汎用量子コンピュータの開発が、対称鍵アルゴリズム・暗号ハッシュ関数・非対称鍵アルゴリズムといった従来の暗号プリミティブにもたらす潜在的なセキュリティリスクについて論じました。また、暗号学的に重要な量子アルゴリズムとそれが影響を与えるプロトコルについても触れました。

量子アルゴリズムの影響が最も顕著なのは非対称鍵暗号の領域であり、Shorのアルゴリズムが素因数分解と離散対数問題に対して多項式時間の解法を提供します。そのため、因数分解と離散対数に基づく非対称暗号システムは、新たな量子安全暗号方式に置き換える必要があります。

これに対して、GroverアルゴリズムおよびBHTアルゴリズムが影響する対称鍵・暗号ハッシュプロトコルでは、量子的な速度向上が超多項式的ではありません。そのため、AESやSHA-256などの既存アルゴリズムは、鍵長・ハッシュ長を十分に確保することで、少なくとも中期的には強化が可能です。

量子安全暗号の基本原理​

計算複雑性​

暗号学において、計算複雑性クラスNP（非決定性多項式時間）は重要な役割を果たします。このクラスは、提示された解を決定性チューリングマシン（DTM）を用いて多項式時間で検証できる決定問題で構成されています。NPの重要性は、古典コンピュータおよび量子コンピュータの両方で効率的に解けないと推測される多くの計算問題を含むとされていることに由来します。

1970年代に開発された最初の世代の成功した非対称鍵暗号システムは、素因数分解や離散対数などの数学的問題にセキュリティの基盤を置いていました。これらは現在、NPのNP中間サブクラスに属すると推測されています。このサブクラスは、DTMでは多項式時間の解法が存在しないと考えられていますが、NPの中で最も困難な問題ほど難しくもない問題で構成されています。

後者はNP完全サブクラスに属します。1990年代のShorのアルゴリズム以降、少なくとも一部のNP中間問題は、DTMではない量子コンピュータ上で効率的に解ける可能性があることが明らかになりました。

そのため、現代の量子安全暗号方式は、NP完全問題または関連するNP困難問題に基づいています。これらの問題は、現時点では量子コンピュータ上でも効率的に解けることが知られていません。

平均ケースと最悪ケースの困難性​

NP困難な問題は数多く知られていますが、すべてが暗号セキュリティの基盤として適しているわけではありません。この文脈では、平均ケース困難性という概念が暗号学上有用です。ある分布からランダムに引き出された問題のインスタンスのほとんどが困難である場合、その問題は平均ケース困難と呼ばれます。一方、一部の孤立した最悪ケースのインスタンスのみが困難な問題は最悪ケース困難と呼ばれます。量子安全暗号の研究者は、平均ケース困難性の仮定を満たす数学的問題を探索し、最悪ケースから平均ケースへの還元などの理論的ツールを活用して、セキュリティと効率が保証される適切なプロトコルを特定しています。

数学的構造​

暗号学者たちは、非対称鍵暗号システムの量子安全な移行に向けた候補として、必要な困難性条件を満たすさまざまな数学的構造を提案しています。主要なファミリーとして以下が知られています：

  1. 格子ベース暗号：このクラスのアルゴリズムは、格子構造上で定義される最短ベクトル問題（SVP）および最近傍ベクトル問題（CVP）などの困難性に依拠しています。代表的な格子ベース方式としてNTRUや学習付き誤り（LWE）が挙げられます。

  2. 符号ベース暗号：この種の暗号は、一般線形符号の復号の困難性に基づいています。最も著名な例としてマクエリース暗号が挙げられます。

  3. 多変数暗号：これらのシステムは、有限体上の多変数方程式を扱います。このカテゴリの有名な方式として、HFE（隠れ体方程式）スキームがあります。

  4. ハッシュベース暗号：暗号ハッシュ関数のみを使用する暗号システムです。マークル署名方式のように、デジタル署名によく利用されます。

  5. 同種写像ベース暗号：これらのシステムは、楕円曲線の代数的構造における特定の問題の困難性に基づいています。超特異同種写像Diffie-Hellman（SIDH）がその一例です。

量子安全暗号のNIST標準化​

量子コンピュータが現在の暗号システムに与え得る影響を認識し、NISTは2016年に量子安全暗号アルゴリズムの標準化プログラムを開始しました。このプロセスは、NISTが2000年代初頭にAES（高度暗号化標準）を標準化した際のプロセスと同様のものです。セキュリティ分野の関係者が参加するオープンで透明なプロセスの中で、複数のQSC候補が提出・評価され、6年間のレビュープロセスを経て、NISTはNISTの量子安全暗号標準に採用される4つのファイナリストを発表しました。

格子ベース暗号​

その名が示すとおり、格子ベース暗号（LBC）は格子と呼ばれる数学的構造上で定義される特定の問題の困難性に基づいています。

格子上の計算問題として基本的に重要なのが、最短ベクトル問題と学習付き誤り問題の2つです。これらについては、以下でいくつかの予備的定義を示した後に詳しく説明します。

格子​

格子とは何かを、身近な世界で考えてみましょう。エッフェル塔（パリ）や鳥の巣スタジアム（北京）のような構造物です。これらは多数の連結された要素を持ち、直線で結ばれた一連の点として捉えることができます。

ダイヤモンドも格子の一形態です。炭素原子が明確で繰り返す構造を持ち、原子結合によって効果的に結びついています。この形成された構造がダイヤモンドの独特な性質を与えます。しかし、異なる結晶構造は非常に異なる配置（格子構造）を持つ可能性があり、その結果として異なる特性を持ちます。このように、格子は暗号学で利用するさまざまな構造の独自の特性を決定する基礎的な数学的フレームワークとして機能します。

数学的に単純に言えば、格子は一定の間隔で規則的に繰り返す点の集合として考えることができます。これらの点が互いにどのように配置されているかを表現するためにベクトルを使います。格子の配置を定義するこれらのベクトルの特定の配列を*基底（basis）*と呼びます。

おもちゃのブロックがたくさん入った箱があり、同じパーツでさまざまなものを作りたいとしましょう。作るものごとに特定の配置が必要で、ブロックを選んで配置する方法が異なるものを作るための基底となります。ブロックの選択や配置を変えると、異なる特性を持つ別のものができあがります。

同様に、格子においても基底は格子の構造を定義する原子的な構成要素のようなものです。格子内のこれらの構成要素（原子や点）をどのように配置するかによって、異なる性質を持つさまざまな格子構造を作ることができます。おもちゃのブロックを変えると作るものが変わるように、基底を変えると格子の特性や性質が変わります。

重要な点として、格子は2次元や3次元に限定されません。高次元に拡張でき、暗号学などの分野では数千以上の次元を扱うことがあります。

最短ベクトル問題​

最短ベクトル問題（SVP）は、格子上の任意の2点間の最短距離（ベクトル）を見つけるという課題です。

最近傍ベクトル問題​

最近傍ベクトル問題（CVP）は、格子上で原点または指定された位置に最も近い点を見つけることを目的とします。

CVPもNP困難であることが知られています。

有界距離復号問題​

CVPと非常に似ているのが有界距離復号（BDD）問題です。

この場合、与えられた原点が格子に近いことはわかっていますが、どれほど近いかは不明です（すなわち有界）。

学習付き誤り問題​

学習付き誤り（LWE）問題は、格子の数学と方程式に誤りを導入するというアイデアを組み合わせたものです。

暗号アルゴリズムの数学的基礎となるいかなる問題においても、一連の計算を行う際に誤りを容易に導入できますが、その逆（誤りを除去して元の解を求めること）は非常に困難です。

LWE問題の注目すべき特徴は、誤りがない場合はガウス消去法などの手法で容易に解ける線形方程式系に帰着するという点です。

しかし、適切な誤り分布を含めることでNP困難な問題になります。

学習付き誤り暗号システムを用いた暗号化​

まず、AliceとBobは問題パラメータに合意します。これらは上記の数学セクションで詳しく説明されていますが、要約すると n,q,N,χn, q, N, \chin,q,N,χ が必要です。

  • qqq は剰余です

誤りを導入する方法も必要です。

ここで χ\chiχ は導入したい誤りを表します。平均0・標準偏差 σ\sigmaσ の離散ガウス分布 Zq\mathbb{Z}_{q}Zq​ 上を使用します。

この誤りの定義も公開情報です。

Pythonでは、標準偏差 σ\sigmaσ の離散ガウス分布 χ\chiχ から引き出されるランダムな誤り/ノイズ寄与 ε\varepsilonε を近似する簡単な関数が必要です。

0付近の分布を確認できるよう、サンプル値をいくつか出力します。

def chi(stdev, modulus):
    return round((np.random.randn() * stdev**2)) % modulus

# print some examples
sd = 2
m = 1000
for x in range(10):
    print("chi = ", chi(sd, m))

次に、Aliceは鍵ペアを生成する必要があります。

彼女は0から qqq の間の nnn 個の値を選択して秘密鍵を設定します。

print(f"Alice's private key: {alice_private_key}")

Aliceは次に、ランダムベクトルを選択し、生成した誤りと組み合わせることで公開鍵を設定します。

数学

より正確には、サンプル As,χA_{s,\chi}As,χ​ を生成する必要があります。

そのために、ランダムベクトル a∈Zqn\mathbf{a} \in \mathbb{Z}_{q}^{n}a∈Zqn​ と χ\chiχ からの誤り ε\varepsilonε を選択して NNN 個のサンプル (a,b)(\mathbf{a}, b)(a,b) を構成します。

# Alice's Public Key
alice_public_key = []

# N is the number of values we want in the key
for i in range(N):
    # Get n random values between 0 and <q
    a = np.random.randint(0, high=q, size=n)
    # get an error to introduce
    epsilon = chi(sigma, q)
    #  calculate dot product (ie like array multiplication)
    b = (np.dot(a, alice_private_key) + epsilon) % q
    # value to be added to the key -
    sample = (a, b)
    alice_public_key.append(sample)

print(f"Alice's public key: {alice_public_key}")

Aliceはこれで公開鍵を共有できます。

そのために、長さ NNN のランダムな2値ベクトル rrr であるマスクを作成します。

# a list of N values between 0 and <2 - ie 0 or 1

このマスクをAliceの公開鍵の対応するエントリに適用し、見つかった値の合計を計算します。

sum_ai = np.zeros(n, dtype=int)
sum_bi = 0

for i in range(N):
    sum_ai = sum_ai + r[i] * alice_public_key[i][0]
    sum_bi = sum_bi + r[i] * alice_public_key[i][1]
sum_ai = [x % q for x in sum_ai]
# sum_bi = sum_bi
ciphertext = (sum_ai, (bob_message_bit * int(np.floor(q / 2)) + sum_bi) % q)

最後に、BobはAliceが秘密鍵を使って復号できる暗号文をブロードキャストします。

# Decryption
adots = np.dot(ciphertext[0], alice_private_key) % q
b_adots = (ciphertext[1] - adots) % q

decrypted_message_bit = round((2 * b_adots) / q) % 2

print(
    f"original message bit={bob_message_bit}, decrypted message bit={decrypted_message_bit}"
)

assert bob_message_bit == decrypted_message_bit

Module-LWEとCRYSTALSスイート​

上記で簡略化された形で紹介し、任意の格子上で一般的に有効なLWE問題は、結果として得られる暗号システムの効率を主に向上させるために、いわゆるRing-LWEフレームワーク内で代数的な環に拡張されています。ただし、Ring-LWEの追加的な代数的構造は潜在的に悪用される可能性があります。現時点では既知のそのような悪用はありませんが。

NISTのQSC標準化プロセスにおける4つのファイナリストのうちの2つ——すなわち、CRYSTALS-Kyber鍵カプセル化メカニズム（KEM）とCRYSTALS-Dilithiumデジタル署名プロトコル——は、モジュール格子と呼ばれる構造および関連するModule-LWEに基づいています。

数学的な詳細については、これらのトピックに関する専門的なコースを参照していただきたいですが、Module-LWEはLWEとRing-LWEの妥協点として見られていることを注記します。LWEよりも効率的（ただしRing-LWEほどではない）でありながら、問題の代数的構造にそれほど依存していないため、Ring-LWEよりも高いレベルのセキュリティ保証を提供します。

鍵カプセル化メカニズムとCRYSTALS-Kyber​

従来の非対称鍵暗号システムは、主に鍵交換とデジタル署名の機能に使用されています。そのため、NISTの標準化プロセスでは、この2つの機能に対して量子安全な代替手段を開発することを目指しました。CRYSTALS-Kyberプロトコルは、RSAのような汎用暗号化スキームではなく、専用の鍵カプセル化メカニズム（KEM）として設計されています。以下に、Kyberを使って2者間で共有秘密を確立するワークフローを示します。

格子ベース暗号におけるIND-CPAとIND-CCAセキュリティ​

対称鍵暗号のレッスンでは、選択平文攻撃に対する識別不能性を指す意味論的セキュリティまたはIND-CPAセキュリティの概念を紹介しました。

対称（AESなど）または非対称（RSAなど）を問わず、従来の暗号システムは決定論的な関数を使って暗号化操作を実装しています。つまり、ある平文を特定の暗号化鍵と組み合わせると、常に同じ暗号文に暗号化されます。このような決定論的暗号システムは、攻撃者が決定論的な暗号化関数に任意の平文の暗号化を要求することで情報を抽出できる選択平文攻撃に対して脆弱です。

このコンテキストでIND-CPAセキュリティを達成するために、暗号化時に初期化ベクトルやパディングを通じて追加の乱数性が導入されます。例えばAESは、ランダムな初期化ベクトルを使用する暗号ブロック連鎖（CBC）またはガロア/カウンタモード（GCM）の動作モードで使用した場合のみIND-CPA安全です。同様に、RSAでもIND-CPAセキュリティを確保するにはOAEPパディングが必要です。

それに対し、格子ベースの暗号化スキームは、問題の定義自体が本質的にランダム化されています。特に、上述のLWEベースの暗号化スキームには、2つの異なる乱数性の要素があります。

  • (1) 分布 χ\chiχ から引き出される誤差（またはノイズ）ε\varepsilonε
  • (2) メッセージの各ビットを暗号化するために使用されるランダムな二進ベクトル r∈{0,1}N\mathbf{r} \in \{0,1\}^Nr∈{0,1}N

誤差 ε\varepsilonε は公開鍵のセキュリティに貢献し、秘密鍵 s\mathbf{s}s を推測することが計算上困難であることを保証します。一方、ランダムな二進ベクトル r\mathbf{r}r は、同じ平文ビットを繰り返し暗号化しても非決定論的になるために必要な本質的な乱数性を提供します。したがって、LWEベースのスキームはパディングなどの外部メカニズムを必要とせずにIND-CPA安全とみなされます。

現代の暗号システムは、選択暗号文攻撃に対する識別不能性を意味するIND-CCAセキュリティの達成を目指しています。この設定では、攻撃者は暗号システムを解読するための情報を抽出することを目的として、自らが選択した非自明な暗号文の集合の復号を取得できます。IND-CCA安全なスキームとは、この能力を持ってしても、攻撃者が暗号化されたメッセージを識別しようとする際にランダムな推測以上の成果を得られないものを指します。IND-CCAはIND-CPAよりも強いセキュリティ概念であり、IND-CPAを包含します。

Kyberなどの量子安全なKEMはIND-CCA安全になるように設計されています。これは2つのステップで達成されます：

  1. IND-CPA安全な公開鍵暗号（PKE）スキームが定義されます。Kyberの場合、そのようなPKEはModule-LWEに基づいています。
  2. 藤崎-岡本変換（FO）の変形を適用して、CCA安全なKEMを得ます。FO変換は、IND-CPA安全な暗号化スキームをIND-CCA安全なものに変換するための汎用的な手法です。詳細については原論文を参照してください。

KyberとDilithiumのセキュリティ機能の詳細、および様々なプログラミング言語でのリファレンス実装については、CRYSTALSスイートのドキュメントを参照することをお勧めします。

ハイブリッド暗号​

ハイブリッド暗号とは、量子安全な公開鍵アルゴリズムと従来の公開鍵アルゴリズム（RSAや楕円曲線など）を組み合わせて使用することで、少なくとも既存の従来型暗号と同等以上のセキュリティを確保するアプローチです。

これにより、次の2つの課題に対処できます：

  1. 既存の標準では、暗号化に使用する特定のアルゴリズムが義務付けられているか、そのようなアルゴリズムが何らかの形で承認されている場合があります。量子安全性を追加するために、まず通常のアルゴリズムで暗号化を行い、その後で量子安全なアルゴリズムによってさらに保護しながら、必要な標準を満たすことができます。

  2. 量子アルゴリズムは新しく、実際に安全で正確であることを確認するためのさらなる分析が必要です。例えば、NISTが当初候補として選定したアルゴリズムの1つが後に解読され、安全でないことが証明されました。このリスクを軽減するため、標準的で十分にレビューされた（量子を除いて！）安全なアルゴリズムと、ポスト量子アルゴリズムの両方を使って暗号化を行うことができます。

まとめ​

非対称鍵暗号システムの量子安全暗号（QSC）への移行は、NP中間問題の古典的な難しさに基づく従来の非対称鍵アルゴリズムを破ることができる量子コンピューターの登場が予想されることによって必要とされています。QSCはNP困難問題に基づいてセキュリティが確立されているため、量子コンピューターを使っても一般的に効率的に解くことができず、量子攻撃に対して堅牢であると推測されています。

この文脈において、誤差あり学習（LWE）問題などの数学的格子理論に根ざした困難な問題が、QSC標準化の有力候補として浮上しています。特に、モジュラー格子に基づくCRYSTALS-KyberとCRYSTALS-Dilithiumアルゴリズムは、RSA、楕円曲線、Diffie-Hellman、DSAなどの一般的な非対称鍵プロトコルに対する近い将来の代替手段として有望です。

量子安全暗号という複雑な状況を乗り越えていく中で、これらの革新的なソリューションを採用することが、量子時代における私たちのデジタルインタラクションの完全性と機密性を守るために重要な鍵となるでしょう。
