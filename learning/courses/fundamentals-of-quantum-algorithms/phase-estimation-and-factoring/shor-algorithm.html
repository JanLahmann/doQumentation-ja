ショアのアルゴリズム

次に、整数因数分解問題に注目し、位相推定を用いて量子コンピュータ上でこの問題を効率的に解く方法を見ていきましょう。 導出されるアルゴリズムが 整数因数分解のためのショアのアルゴリズム です。 ショアは自身のアルゴリズムを特に位相推定の言葉で説明していませんでしたが、その仕組みを説明するうえで自然かつ直感的なアプローチです。

まず、位数発見問題 と呼ばれる中間問題について説明し、位相推定がこの問題の解を与えることを確認します。 次に、位数発見問題の効率的な解が整数因数分解問題の効率的な解をもたらすことを見ていきます。 （ある問題の解が別の問題の解を与えるとき、後者の問題は前者に 帰着する と言います。ここでは整数因数分解を位数発見に帰着しています。） ショアのアルゴリズムのこの第二部分は量子計算をまったく使いません。完全に古典的です。 量子計算が必要なのは位数発見を解くためだけです。

位数発見問題​

初等整数論の基礎​

位数発見問題と、それを位相推定で解く方法を説明するために、まずいくつかの基本的な整数論の概念を確認し、便利な記法を導入しておきましょう。

はじめに、任意の正整数 NNN に対して、集合 ZN\mathbb{Z}_NZN​ を次のように定義します。

ZN={0,1,…,N−1}\mathbb{Z}_N = \{0,1,\ldots,N-1\}ZN​={0,1,…,N−1}

例えば、 Z1={0},  \mathbb{Z}_1 = \{0\},\;Z1​={0}, Z2={0,1},  \mathbb{Z}_2 = \{0,1\},\;Z2​={0,1}, Z3={0,1,2},  \mathbb{Z}_3 = \{0,1,2\},\;Z3​={0,1,2}, などとなります。

これらは数の集合ですが、単なる集合以上のものとして考えることができます。 特に、ZN\mathbb{Z}_NZN​ 上の 演算（加算や乗算など）を考えることができ、答えを常に NNN で割った余り（すなわち NNN を法とした剰余）として取ることで、これらの演算を施しても常にこの集合の中に留まります。 NNN を法とした加算と乗算という二つの演算によって、ZN\mathbb{Z}_NZN​ は代数において基本的に重要なオブジェクトである 環 になります。

例えば、333 と 555 は Z7\mathbb{Z}_7Z7​ の元であり、これらを掛け合わせると 3⋅5=153\cdot 5 = 153⋅5=15 となり、これを 777 で割ると余りが 111 になります。 これを次のように表すことがあります。

3⋅5≡1  (mod 7)3 \cdot 5 \equiv 1 \; (\textrm{mod } 7)3⋅5≡1(mod 7)

位相推定による位数発見​

与えられた a∈ZN∗a\in\mathbb{Z}_N^{\ast}a∈ZN∗​ に対する位数発見問題を解くには、演算 MaM_aMa​ に位相推定手順を適用します。

そのためには、MaM_aMa​ だけでなく Ma2M_a^2Ma2​、Ma4M_a^4Ma4​、Ma8M_a^8Ma8​ など、位相推定手順から十分精度の高い推定値を得るために必要なだけ量子回路で効率的に実装する必要があります。 ここではその方法を説明し、必要な精度の度合いについては後ほど詳しく確認します。

まず MaM_aMa​ 単体の演算から始めましょう。 量子回路モデルで作業しているため、当然ながら 000 から N−1N-1N−1 までの数を2進法で符号化します。 符号化すべき最大の数は N−1N-1N−1 なので、必要なビット数は

したがって、{∣ψ0⟩,…,∣ψr−1⟩}\{\vert\psi_0\rangle,\ldots,\vert\psi_{r-1}\rangle\}{∣ψ0​⟩,…,∣ψr−1​⟩} が正規直交集合であるという事実から、上位 mmm 量子ビットの測定により、k∈{0,…,r−1}k\in\{0,\ldots,r-1\}k∈{0,…,r−1} が一様ランダムに選ばれた値 k/rk/rk/r に対する近似値 y/2my/2^my/2m が得られることがわかります。 すでに述べたように、これにより複数回の独立した実行を経て高い確信度で rrr を求めることができ、これが目標でした。

総コスト​

各制御ユニタリ MakM_a^kMak​ を実装するコストは O(n2)O(n^2)O(n2) です。 制御ユニタリ演算は mmm 個あり、m=O(n)m = O(n)m=O(n) ですので、制御ユニタリ演算の総コストは O(n3)O(n^3)O(n3) となります。 さらに、mmm 個のアダマールゲート（コストへの寄与は O(n)O(n)O(n)）と、O(n2)O(n^2)O(n2) を寄与する逆量子フーリエ変換があります。 したがって、制御ユニタリ演算のコストが手順全体のコストを支配し、全体コストは O(n3)O(n^3)O(n3) となります。

量子回路自体に加えて、途中でいくつかの古典的計算も必要です。 これには、制御ユニタリゲートの作成に必要な ZN\mathbb{Z}_NZN​ における冪 aka^kak（k=2,4,8,…,2m−1k = 2, 4, 8, \ldots, 2^{m-1}k=2,4,8,…,2m−1）の計算、および θ\thetaθ の近似値を分数に変換する連分数アルゴリズムが含まれます。 これらの計算は、総コスト O(n3)O(n^3)O(n3) のブール回路で実行できます。

通常のケースと同様に、漸近的に高速なアルゴリズムを使用することでこれらの上限をすべて改善できます。上記の上限は基本的な算術演算に標準的なアルゴリズムを使用していると仮定しています。

位数発見による素因数分解​

最後に議論すべきことは、位数発見問題を解くことが素因数分解にどのように役立つかです。 この部分は完全に古典的なものであり、量子計算とは直接関係しません。

基本的な考え方は以下のとおりです。 数 NNN を素因数分解したいと考えており、これを再帰的に行うことができます。 具体的には、NNN の分割（任意の2つの整数 b,c≥2b,c\geq 2b,c≥2 で N=bcN = bcN=bc を満たすものを見つけること）に集中できます。 NNN が素数の場合はこれが不可能ですが、まず素数判定アルゴリズムで NNN が素数かどうかを効率的に確認でき、素数でなければ分割を試みます。 NNN を分割したら、bbb と ccc に対して再帰を繰り返し、すべての因数が素数になって NNN の素因数分解が得られるまで続けます。

偶数の分割は簡単で、222 と N/2N/2N/2 を出力するだけです。

完全冪数、すなわち整数 s,j≥2s,j\geq 2s,j≥2 に対して N=sjN = s^jN=sj の形の数の分割も容易です。 N1/2,N^{1/2},N1/2, N1/3,N^{1/3},N1/3, N1/4N^{1/4}N1/4 などの根を近似し、sss の候補として近傍の整数を確認することで行えます。 根が 222 を下回って追加の候補が現れなくなるため、この数列の log⁡(N)\log(N)log(N) ステップ先まで調べれば十分です。

これらの両方ができることは重要です。位数発見は偶数の因数分解や、sss が素数である素数冪の因数分解には役立たないからです。 しかし、NNN が奇数かつ素数冪でない場合は、位数発見を使って NNN を分割することができます。

素数冪でない奇数の合成数 N を分割する確率的アルゴリズム

  1. a∈{2,…,N−1}a\in\{2,\ldots,N-1\}a∈{2,…,N−1} をランダムに選ぶ。

  2. d=gcd⁡(a,N)d=\gcd(a,N)d=gcd(a,N) を計算する。

  3. d>1d > 1d>1 ならば b=db = db=d と c=N/dc = N/dc=N/d を出力して終了する。そうでなければ a∈ZN∗a\in\mathbb{Z}_N^{\ast}a∈ZN∗​ であることがわかっているので次のステップへ進む。

  4. rrr を aaa の NNN に対する位数とする（ここで位数発見が必要になります）。

  5. rrr が偶数の場合：

    5.1 x=ar/2−1x = a^{r/2} - 1x=ar/2−1（NNN を法として）を計算する
    5.2 d=gcd⁡(x,N)d = \gcd(x,N)d=gcd(x,N) を計算する。
    5.3 d>1d>1d>1 ならば b=db=db=d と c=N/dc = N/dc=N/d を出力して終了する。

  6. この地点に達した場合、アルゴリズムは NNN の因数を見つけることに失敗しています。

このアルゴリズムの実行は NNN の因数を見つけることに失敗する場合があります。 具体的には、次の2つの状況で失敗します。

  • aaa の NNN に対する位数が奇数である。
  • aaa の NNN に対する位数が偶数であり、gcd⁡(ar/2−1,N)=1\gcd\bigl(a^{r/2} - 1, N\bigr) = 1gcd(ar/2−1,N)=1 である。

基本的な整数論を用いると、ランダムな aaa の選択に対して、少なくとも 1/21/21/2 の確率でこれらのいずれの事象も起きないことが証明できます。 実際、どちらかの事象が起きる確率は 2−(m−1)2^{-(m-1)}2−(m−1) 以下であり、ここで mmm は NNN の異なる素因数の数です。 これが NNN が素数冪でないという仮定が必要な理由です。 （NNN が奇数であるという仮定も、この事実が成立するために必要です。）

つまり、各実行で NNN を分割できる確率は少なくとも 50% です。 したがって、aaa をそれぞれランダムに選びながらアルゴリズムを ttt 回実行すれば、少なくとも 1−2−t1 - 2^{-t}1−2−t の確率で NNN の分割に成功します。

アルゴリズムの基本的な考え方は次のとおりです。 位数 rrr が偶数となる aaa の選択があれば、r/2r/2r/2 は整数となり、次の数を考えることができます。

ar/2−1  (mod  N)andar/2+1  (mod  N).a^{r/2} - 1\; (\textrm{mod}\; N) \quad \text{and} \quad a^{r/2} + 1\; (\textrm{mod}\; N).ar/2−1(modN)andar/2+1(modN).

公式 Z2−1=(Z+1)(Z−1)Z^2 - 1 = (Z+1)(Z-1)Z2−1=(Z+1)(Z−1) を使うと、

(ar/2−1)(ar/2+1)=ar−1\bigl(a^{r/2} - 1\bigr) \bigl(a^{r/2} + 1\bigr) = a^r - 1(ar/2−1)(ar/2+1)=ar−1

が導かれます。

位数の定義から ar  (mod  N)=1a^r \; (\textrm{mod}\; N) = 1ar(modN)=1 であることがわかります。つまり NNN は ar−1a^r - 1ar−1 を整除するということです。 したがって NNN は次の積を整除します。

(ar/2−1)(ar/2+1).\bigl(a^{r/2} - 1\bigr) \bigl(a^{r/2} + 1\bigr).(ar/2−1)(ar/2+1).

これが成り立つためには、NNN のすべての素因数が ar/2−1a^{r/2} - 1ar/2−1 または ar/2+1a^{r/2} + 1ar/2+1（あるいは両方）の素因数でなければなりません。ランダムな aaa の選択では、NNN のすべての素因数が一方の項のみを割り切り、もう一方は割り切らないことはほぼ起こりません。 そうでなければ、NNN の素因数の一部が第一項を割り切り、残りが第二項を割り切る限り、第一項との GCD を計算することで NNN の非自明な因数を見つけることができます。
